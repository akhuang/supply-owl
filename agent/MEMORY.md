## 决策原则 (Axioms)

### 温度判断（CPD vs RPD + 齐套 + 外部事件）

- CPD 未承诺 + 供应经理催了 = 🔴 红色，当天催承诺，下班前做到位
- CPD 未承诺 + 没人催 = 不主动盯
- CPD 不满足 RPD（CPD > RPD）+ 差距 ≥ 5 天 = 🟡 黄色，持续提拉
- CPD 不满足 RPD（CPD > RPD）+ 差距 < 5 天 = 🟡 黄色，关注
- CPD 满足 RPD 但一线/供应经理仍要求提拉 = 按实际情况判断（不自动降温）
- ETA > CRD（客户交期风险）= 🔴 红色
- 欠料 = 🟡 黄色，需关注齐套
- 急单 = 温度 +1 级
- 客户/内部 24 小时内催过 = 温度 +1 级
- 3 天没有任何操作的合同 = 🟡 黄色，检查是否遗漏

### "找谁"路由规则

- CPD 未承诺 + 供应经理催了 → 找**承诺坐席**
- CPD 不满足 RPD + 批次 HWA 开头 → 直接用 **1Y0 合同**找**大调度**
- CPD 不满足 RPD + 批次有关联 00E 合同 → 用 **00E 合同**找**大调度**
- CPD 不满足 RPD + 其它 → 找**提拉座席**

### 处理优先级

- 红色先于黄色，黄色先于绿色
- 同等温度下，有人催的优先
- 多批次合同按"该找谁"分组处理，不按批次逐个处理

### 行动模式

- CPD 未承诺 + 供应经理催了 → 催承诺坐席（WeLink 先催，后电话）
- CPD 不满足 RPD → 找大调度/提拉座席持续提拉
- CPD 满足 RPD 但一线要求提拉 → 仍需找大调度沟通
- CPD 满足 RPD 且无额外要求 → 正常跟踪
- 多个大调度/座席 → 批量发邮件，不要一个个找
- 收到回复 → 汇总后统一回复请求人

### 提拉要素

- 提拉必须有**目标日期**（至少满足 RPD）
- 付款是前置条件：CPD - 8 天必须付款
- 付款是供应经理的事，先确认付款能力再定目标
- 给大调度的消息不写付款信息，只写目标日期
- 下午了基本当天付不了，最早明天 → 目标日期 = 明天+8天
